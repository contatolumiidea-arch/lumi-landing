# LUMI ONBOARDING — Client Folder Structure

Each client who completes the onboarding wizard receives a unique `client_id` (UUID).
After the wizard is complete, create a folder for that client using the structure below.

## Folder Structure

```
clients/
└── {client_id}/
    ├── config.json              ← Configuration file generated by the onboarding wizard
    ├── assets/
    │   ├── logo.png             ← Client logo (uploaded during Step 3)
    │   ├── profile.jpg          ← Realtor profile photo (uploaded during Step 3)
    │   └── properties/
    │       ├── prop-01.jpg      ← Property photo 1 (uploaded during Step 3)
    │       ├── prop-02.jpg      ← Property photo 2
    │       ├── prop-03.jpg      ← Property photo 3
    │       ├── prop-04.jpg      ← Property photo 4
    │       └── prop-05.jpg      ← Property photo 5 (max)
    ├── translations/
    │   ├── pt.json              ← Custom Portuguese overrides (optional)
    │   ├── en.json              ← Custom English overrides (optional)
    │   └── es.json              ← Custom Spanish overrides (optional)
    └── settings/
        └── integrations.json   ← CRM webhook URLs and integration tokens
```

## Status Values

| Status | Meaning |
|--------|---------|
| `pending_setup` | Onboarding complete, awaiting LUMI IDEA team setup |
| `in_progress` | Team is building the landing page |
| `review` | Page ready for client review |
| `active` | Landing page is live |
| `paused` | Page temporarily offline |

## Notes

- The `client_id` is generated as a UUID v4 during the onboarding wizard.
- Asset files uploaded during the wizard must be manually transferred to this folder structure.
- The `translations/` folder is optional and only needed when clients request custom text overrides beyond the default translations.
- Keep `integrations.json` secure — it contains webhook URLs and API tokens.
